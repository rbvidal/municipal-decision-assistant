## Version 1.1 Semantic Refinement

The following semantic clarifications were added without changing navigation, APIs, workflows, or layouts:

| Refinement | What Changed | Location |
|---|---|---|
| **Tasks as a first-class business object** | Aufgabe (Task) expanded from 3 lines to a full treatment. Defines 9 typical tasks generated by a Vorgang through its lifecycle. Every task has owner, due date, status, priority, and related case. Clarifies that the Home screen is fundamentally a task queue, not merely a case list. | Section 2 (Core Business Objects) |
| **Knowledge vs. Documents distinction** | New subsection explains that Wissen (Knowledge) is consulted — regulations, procedures, templates, FAQs. Dokumente (Documents) are managed — uploaded PDFs, submissions, generated decisions. The UI for each area serves fundamentally different user needs. | Section 2 (Core Business Objects) |
| **Home page reframed as Operational Workbench** | Startseite description replaced. The Home page is not a dashboard — it is an Operational Workbench whose primary purpose is answering "What should I work on next?" Work queues and deadlines have highest visual importance. Statistics are tertiary. | Section 5.3 (Startseite) |
| **Visual Priority hierarchy** | New subsection defines a 7-level visual emphasis hierarchy: P1 Current Work → P2 Urgent Deadlines → P3 Waiting Items → P4 Approvals → P5 Decision Support → P6 Statistics → P7 Administration. Every screen must make the P1 element identifiable within 500ms. | Section 9 (Visual Design) |
